01   Overview

How the system
detects & reacts

This project implements a real-time obstacle detection system using an infrared sensor module (FC-51) paired with an Arduino Uno microcontroller. When an object enters the sensing range, the system immediately triggers multi-modal alerts.

The FC-51 IR module emits modulated infrared light through its built-in IR LED. The integrated photodiode receiver picks up the reflected signal when an obstacle is present — pulling the digital output pin to LOW (active-low logic).

The Arduino firmware reads this signal every 200 milliseconds and activates a passive buzzer, an LED indicator, and updates a 16×2 I2C LCD with the current status — either "Obstacle Detected!" or "No Obstacle."

05   Source Code

Arduino Firmware

obstacle_detection.ino 
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

// ─── Pin Definitions ──────────────────────────────────────
#define IR_SENSOR 7    // FC-51 digital output → Arduino D7
#define BUZZER    8    // Passive buzzer positive → D8
#define LED       13   // LED anode (via 220Ω resistor) → D13

// ─── LCD Initialisation (I2C address 0x27, 16 cols, 2 rows)
LiquidCrystal_I2C lcd(0x27, 16, 2);

// ─── Setup ────────────────────────────────────────────────
void setup() {
  pinMode(IR_SENSOR, INPUT);
  pinMode(BUZZER,    OUTPUT);
  pinMode(LED,       OUTPUT);

  // Initialise and display splash message on LCD
  lcd.init();
  lcd.backlight();
  lcd.setCursor(0, 0);
  lcd.print("IR Obstacle Alert");
  delay(2000);
  lcd.clear();
}

// ─── Main Loop ────────────────────────────────────────────
void loop() {
  // Most FC-51 modules are active-LOW: LOW = obstacle present
  int obstacle = digitalRead(IR_SENSOR);

  if (obstacle == LOW) {
    // ── Obstacle detected ──────────────────────────────────
    lcd.setCursor(0, 0);
    lcd.print("Obstacle Detected!");
    digitalWrite(BUZZER, HIGH);   // Buzzer ON
    digitalWrite(LED,    HIGH);   // LED ON
  } else {
    // ── Path clear ────────────────────────────────────────
    lcd.setCursor(0, 0);
    lcd.print("No Obstacle      ");  // trailing spaces clear old text
    digitalWrite(BUZZER, LOW);    // Buzzer OFF
    digitalWrite(LED,    LOW);    // LED OFF
  }

  delay(200);  // 200 ms loop delay — avoids LCD flicker
}
06   Principle

How it works

01
Infrared Emission

The FC-51 module's built-in IR LED continuously emits modulated infrared light at ~850 nm. This invisible beam projects forward from the sensor face, creating a detection zone adjustable via the onboard potentiometer.

02
Reflection & Detection

When a solid object enters the sensing range, the IR light reflects back toward the module. The integrated photodiode receiver detects this reflected energy. The onboard LM393 comparator evaluates the signal strength against a threshold set by the potentiometer.

03
Digital Signal Output

The comparator drives the OUT pin LOW when an obstacle is detected (active-low logic) and HIGH when the path is clear. This clean digital signal is read directly by the Arduino on digital pin D7 using digitalRead().

04
Alert Activation

Upon detecting a LOW signal, the Arduino firmware simultaneously drives D8 HIGH (buzzer), D13 HIGH (LED), and writes "Obstacle Detected!" to the I2C LCD. All three alerts deactivate instantly when the obstacle is removed — with a 200 ms polling loop to prevent display flicker.
